EXPLICACION DE PARTE INVERSA 

Estructura inversa: Diccionario de listas

En el código anterior usamos una estructura tipo “diccionario de diccionarios”, donde cada clave es el ID del empleado y el valor son sus datos.

Tiene la siguiente pinta: empleados = {
    "E01": {"nombre": "Juan", "rol": "mecánico", "horas": 42},
    "E02": {"nombre": "Ana", "rol": "camarero", "horas": 38}
}

Pero ahora queremos el caso inverso qué es el siguiente 

Estructura inversa: “diccionario de listas”

Tiene la siguiente pinta : 

empleados_inverso = {
    "id": ["E01", "E02"],
    "nombre": ["Juan", "Ana"],
    "rol": ["mecánico", "camarero"],
    "horas": [42, 38]
}

QUE DIFERENECIAS PODEMOS ENCONTRAR

Aspecto     	        Diccionario de diccionarios         	              Diccionario de listas

Organización	        Cada clave representa un empleado completo  	      Cada lista almacena un campo de todos los empleados

Ideal para  	        Acceder a los datos de un empleado concreto	          Analizar o recorrer un campo en todos los empleados

Ejemplo de acceso   	empleados["E01"]["rol"] → "mecánico"	              empleados_inverso["rol"][0] → "mecánico"

Usos comunes	        CRUD (crear, leer, actualizar, eliminar registros)	  Estadísticas o procesamiento masivo de columnas
